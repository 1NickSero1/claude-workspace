========================================
  PALABRAS CLAVE — Comandos por voz de texto
  Definidas hasta 2026-07-02
========================================

Estas palabras se pueden usar en cualquier momento de la conversación
con Claude Code para disparar una acción automática, sin tener que
explicar el proceso completo cada vez.

------------------------------------------------------------
COMMITS DE GIT (definidas en CLAUDE.md raíz)
------------------------------------------------------------

TERMINAR
  → Commit GENERAL: agrega y confirma TODOS los cambios pendientes
    del repo hasta ese momento, con mensaje que resume qué se hizo.

PAUSA
  → Commit ESPECÍFICO: Claude primero pregunta cuál actualización o
    archivo se quiere confirmar, y hace commit solo de eso.

SUBIR
  → Push automático: ejecuta "git push" de inmediato, sin preguntar.
    Esta SÍ es un hook real (UserPromptSubmit, .claude/hooks/subir-push.ps1).
    Si falla (rama desactualizada, conflicto), Claude reporta el error
    y no intenta resolverlo por su cuenta.

------------------------------------------------------------
CAMBIO DE PROYECTO (hook real)
------------------------------------------------------------

WALLET CONTROL
  → Invoca la skill PECAS y saluda, recomendando correr /clear
    antes de seguir, para arrancar enfocado en APPS/wallet-control.

RUTA SEGURA
  → Igual, pero enfocado en APPS/ruta-segura.

  Nota: Claude no puede borrar su propio contexto de conversación
  desde un hook — eso solo lo dispara el usuario con /clear.
  El hook recomienda hacerlo, no lo fuerza.

------------------------------------------------------------
SERVIDORES DE DESARROLLO
------------------------------------------------------------

"abre el servidor" (contexto wallet-control)
  → Ejecuta directo: npx expo start --port 8082 en APPS/wallet-control

"abre el servidor" (contexto ruta-segura)
  → Ejecuta directo: npx expo start --port 8082 en APPS/ruta-segura

------------------------------------------------------------
ENTRENAMIENTO DE CLAUDE.md (raíz del sistema)
------------------------------------------------------------

"soy experto en [tema]"
  → Agrega el tema a la tabla de Expertise

"elimina expertise [tema] de CLAUDE.md"
  → Quita el tema de la tabla de Expertise

"agrega [nombre] a proyectos activos"
  → Registra un proyecto nuevo en la tabla de Proyectos Activos

"mueve [nombre] a proyectos terminados"
  → Pasa un proyecto de Activos a Terminados

"agrega lección: [lo que aprendiste]"
  → Agrega una fila a la tabla de Lecciones Aprendidas

"actualiza CLAUDE.md con [instrucción]"
  → Cualquier otro cambio libre al archivo raíz

------------------------------------------------------------
COMANDO DE ESTE PROYECTO
------------------------------------------------------------

MEJORAX
  → Lee NOTAS/ajustes wallet-control.txt y resuelve todo lo
    pendiente en la sección [PENDIENTE]

------------------------------------------------------------
HOOKS REALES ACTIVOS (.claude/settings.local.json)
------------------------------------------------------------

SUBIR → UserPromptSubmit → .claude/hooks/subir-push.ps1
WALLET CONTROL / RUTA SEGURA → UserPromptSubmit → .claude/hooks/proyecto-switch.ps1

[sin palabra clave, se disparan solos por evento]
  → PreToolUse Edit/Write: bloquea archivos *metricas*.txt (NO MODIFICAR)
  → PreToolUse Read/Edit/Write: bloquea .env y .env.*
  → SessionStart: muestra [PENDIENTE] de NOTAS/ajustes wallet-control.txt
  → PostToolUse tras "git commit": avisa si quedó sin pushear

========================================
  Nota: TERMINAR y PAUSA son reglas de comportamiento, no hooks
  de sistema (no están en settings.json). Claude las sigue porque
  están documentadas en CLAUDE.md raíz. SUBIR sí es un hook técnico.
========================================
