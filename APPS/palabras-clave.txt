PALABRAS CLAVE DEL SISTEMA
==========================
Indice de todas las palabras clave creadas para Claude Code en este repo, con su significado
resumido en una palabra y si aplican en general o estan atadas a una skill/proyecto especifico.

Este es el UNICO archivo de indice de palabras clave del repo (antes existian copias parciales
en APPS/wallet-control/NOTAS/palabras-clave.txt y APPS/ruta-segura/NOTAS/palabras-clave.txt;
se eliminaron el 2026-07-09 para no mantener 3 archivos desincronizados).

Este archivo se actualiza cada vez que se crea una palabra clave nueva (con o sin hook real
detras). La fuente de verdad de COMO funciona cada una varia segun el tipo:
- Generales y las que tienen hook real -> CLAUDE.md (secciones "Palabra(s) Clave de ...")
- MEJORAX -> SKILLS/FIXA.md
- "abre el servidor" / "actualiza el plan" -> RECETAS/receta-apps.txt
Aqui solo va el resumen rapido de una linea por palabra.

--------------------------------------------------------------------------------------------------------------
PALABRA CLAVE          | SIGNIFICADO (1 palabra) | TIPO                        | DONDE APLICA
--------------------------------------------------------------------------------------------------------------
TERMINAR               | Commitear               | General                     | Cualquier proyecto (git)
PAUSA                  | Parcial                 | General                     | Cualquier proyecto (git)
SUBIR                  | Pushear                 | General                     | Cualquier proyecto (git)

    Pendiente decidir si se documenta formalmente en algun archivo del proyecto.

Notas sobre los TIPOS:
- "General" = no depende de ninguna skill ni proyecto puntual, gobierna una accion transversal
  (en este caso, siempre relacionada a git: commit o push).
- "Especifica de proyecto" = siempre apunta al mismo proyecto fijo, sin importar el contexto de
  la conversacion (no pregunta cual, y no cambia si estas hablando de otro proyecto).
- "Especifica de skill" = siempre activa la misma skill, pero esa skill pregunta cual proyecto de
  APPS/ usar (no esta atada a uno solo).
- "Contextual (proyecto activo)" = el comportamiento es el mismo patron en cualquier proyecto,
  pero actua sobre EL PROYECTO QUE ESTES TRABAJANDO EN ESE MOMENTO segun el contexto de la
  conversacion (ej. si veniste de decir WALLET CONTROL, actua sobre wallet-control; si veniste de
  RUTA SEGURA, actua sobre ruta-segura). No tiene un hook real que lo detecte solo — depende de
  que el contexto de la conversacion ya deje claro en cual proyecto estas.

Detalle de cada una:
- TERMINAR: commit general - agrega y confirma TODO lo pendiente en el repo.
- PAUSA: commit especifico - Claude pregunta primero que actualizacion/archivo confirmar.
- SUBIR: push automatico e inmediato de lo ya commiteado (hook real, sin preguntar).
- WALLET CONTROL: hook real - hace git pull, invoca PECAS y recomienda /clear, enfocado en
  APPS/wallet-control.
- RUTA SEGURA: igual que WALLET CONTROL pero enfocado en APPS/ruta-segura.
- PDF AUDITA: hook real - corre la skill AUDITA completa y entrega el informe directo en PDF en
  APPS/<proyecto>/AUDITORIAS/, preguntando solo cual proyecto.
- IMAGINA: hook real - corre la skill ESTETIK y entrega un mini PDF solo con los cambios/mejoras
  visuales recomendadas en APPS/<proyecto>/PDF/, preguntando solo cual proyecto.
- MEJORAX: sin hook real (regla de comportamiento, definida en SKILLS/FIXA.md) - lee el archivo
  de ajustes del proyecto activo (NOTAS/ajustes wallet-control.txt o NOTAS/ajustes app ruta.txt),
  resuelve todo lo que este en [PENDIENTE], lo mueve a [RESUELTO], y agrega una entrada nueva en
  el changelog del proyecto (ACTUALIZACIONES.txt en ruta-segura).
- "abre el servidor" / "abrir el servidor": sin hook real formal (aunque hay permisos
  preconfigurados en .claude/settings.local.json) - corre "npx expo start --port 8082" en la
  carpeta del proyecto activo.
- "actualiza el plan": solo documentada para wallet-control - regenera PDF/wallet-control-avance.pdf
  via Puppeteer con el sistema visual fijo de la app, no el plan interno de Claude Code.
- SUBIR PROYECTO: segun la memoria de Claude, en wallet-control dispara
  "eas build --platform android --profile preview" - distinto de SUBIR (que es git push). No
  confirmado por escrito en ningun archivo del repo todavia.
